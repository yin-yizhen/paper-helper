# paper-helper

`paper-helper` 是一个面向中文大学结课论文的 Codex skill。它会把“选题—知网核验—写作—引文检查—Word 交付”组织成一条可追踪流程。

## 先说结论：是不是一键？

**可以一键启动，但不是绕过知网限制的无人值守机器人。**

在环境齐全时，你只需发一句话，例如：

```text
用 $paper-helper 写一篇“城中村更新中的社区治理”的 6000 字课程论文，中文参考文献必须在知网可查。
```

它会自行询问缺失的少量关键选项，并依次完成选题细化、CNKI 题录核验、写作、引文台账和 Word 交付。

但以下操作必须由你在浏览器里完成一次：CNKI 登录、验证码、机构授权、付费确认。这些是网站访问控制，skill 不会也不应该绕过。完成后告诉 Codex“继续”，它会从已保存的台账继续，而不是从头再来。

| 能力 | 环境齐全时 | 缺少条件时 |
| --- | --- | --- |
| 选题、提纲、原创初稿 | 自动完成 | 自动完成 |
| CNKI 题录与期刊等级核验 | 自动检索、逐条记录 | 明确标记“未核验” |
| 最终引用文献下载 | 自动触发并记录下载 | 等待你登录/授权，或标记无权限 |
| 引文与文末文献闭环 | 自动检查 | 自动检查 |
| Word 生成与逐页视觉检查 | 自动生成并在有渲染器时检查 | 生成后明确标记“未完成视觉验收” |

它绝不会把“未核验”“未下载”“无权限”写成已完成，也不会伪造参考文献、数据或查重结果。

## 它适合什么

- 3000—8000 字左右的中文结课论文
- 文献综述、理论分析、案例分析、规划/策略类论文
- 要求中文参考文献可在 CNKI 检索、优先核心期刊的任务
- 希望直接拿到 `.docx`、文献核验报告和引文一致性检查报告的任务

没有用户提供的真实数据时，它不会虚构问卷、访谈、统计结果或案例成效来写“实证论文”。

## 安装

### 已有 Codex 的用户

```powershell
git clone https://github.com/yin-yizhen/paper-helper.git
Copy-Item -Recurse .\paper-helper "$env:CODEX_HOME\skills\paper-helper"
```

如果没有设置 `CODEX_HOME`，复制到：

```text
%USERPROFILE%\.codex\skills\paper-helper
```

然后重启或刷新 Codex。

### 第一次使用前的三项检查

要获得“从主题到可交付 Word”的完整体验，需要：

1. **可见 Chrome 与 CNKI 能力**：Codex 必须能打开/控制用户看得见的 Chrome，或环境中已有可绑定到该 Chrome 的 `cnki-search`、`cnki-paper-detail`、`cnki-journal-index`、`cnki-download` 等能力。CNKI 不应在后台浏览器里静默执行。
2. **CNKI 访问权限**：题录检索通常可以直接进行；下载全文需要你自己的 CNKI 账号或学校/机构权限。
3. **Word 能力**：需要可用的 DOCX 创建环境；若要逐页视觉验收，还需要 LibreOffice 或等价渲染后端。

**只安装本仓库，并不会自动给别人的 Codex 安装上述 Chrome 控制能力、CNKI 权限或 Word 渲染器。**首次运行时，skill 会进行预检并告诉你缺什么；缺少可见 Chrome 控制时仍能写提纲和待核验草稿，但不能声称文献已经核验。

如果 Chrome 出现验证码、登录页、机构授权页或 `ERR_CERT_*` 证书错误，skill 会把窗口保留在当前页面，让你手动处理后再继续；它不会在后台浏览器里绕开这个页面。

## 最简单的使用方式

只给主题也可以：

```text
用 $paper-helper 写“乡村振兴背景下县域城乡融合发展”的课程论文。
```

它只会补问必要的信息，通常是：字数、论文类型、文献层级。你不需要先自己找参考文献。

如果课程有格式模板，直接把 `.docx` 拖进对话并说“按这个模板交付”。PDF 可以作为版式参考，但不适合直接复用 Word 样式。

## 你会拿到什么

- `论文题目.docx`：题目、摘要、关键词、正文和参考文献。
- `references.json`：每篇最终文献的 CNKI 链接、核验状态、期刊索引状态、下载状态和正文引用位置。
- `文献核验结果.md`：方便你逐条复查的文献台账。
- `引文一致性检查.md`：检查正文 `[n]` 与文末条目是否闭环。

最终交付说明还会列出实际字数、已核验篇数、已下载篇数、未下载原因，以及 Word 是否已完成视觉验收。

## 边界

- 不绕过 CNKI 登录、验证码、付费或机构权限。
- 不承诺知网、维普、Turnitin 的具体相似率，也不承诺 AI 检测或学校最终审核结果。
- 不通过复制原文、拼贴改写或刻意重复来追求某个查重数值。
- 不要把密码、Cookie、验证码或受版权保护的全文提交到这个仓库。
