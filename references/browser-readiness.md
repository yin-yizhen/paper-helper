# 浏览器就绪门

在任何 CNKI 检索前，先将当前状态归入以下一种；不要根据用户的“继续”或“已打开”猜测。

| 状态 | 可做的事 | 下一步 |
| --- | --- | --- |
| 已连接且页面正常 | 检索、提取题录、下载 | 继续 CNKI 工作流 |
| 浏览器已手动打开但未连接 | 选题、提纲、待核验草稿 | 说明当前 Codex 无法读取该窗口；请连接 Chrome 扩展或使用 Codex 应用内浏览器 |
| 418 / 证书错误 / 验证码 / 登录页 | 保存已完成台账 | 让用户先在可见页面完成处理；不写“已核验” |
| 无浏览器能力 | 选题、提纲、待核验草稿 | 明确说明无法进行 CNKI 核验 |

判定“页面正常”的最低证据是：可读取 `input.search-input` 检索框或 `.result-table-list tbody tr` 结果行。没有这些证据时，不得启动题录抓取。

不要使用后台 Playwright、裸 HTTP 请求、`Start-Process`、`cmd start` 或证书忽略参数来跨过此门槛。Microsoft Edge 仅在当前 Codex 明确提供 Edge/桌面应用控制时可被自动接管；用户手动打开 Edge 本身不是连接证据。
