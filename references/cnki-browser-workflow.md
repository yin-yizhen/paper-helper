# CNKI 浏览器操作

CNKI 必须在用户可见且已连接的浏览器标签完成。标签取得顺序、验证码判定和硬停止条件见 [浏览器就绪门](browser-readiness.md)。优先调用已安装的 CNKI 专用 skills；本文件是页面变化时的稳健回退流程。

## 检索与筛选

1. 先获取可访问性快照，按可访问名称/角色定位检索框和“检索”按钮；CSS 只作回退。任何点击/输入前都确认候选唯一，若有多个先缩小到检索区域。
2. 检索框回退顺序可为 `input.search-input`、可访问名称含“检索/主题”的 textbox。按钮回退顺序为可访问角色 button/name=检索、`input.search-btn`。某个选择器不存在时换下一条，不要宣告 CNKI 不可用。
3. 每次修改关键词、切换检索模式或翻页后，都重新应用用户要求的 CSSCI/北大核心/CSCD、年份和文献类型条件；随后读取已选筛选项/标签及结果计数，确认条件没有被 CNKI 重置。未确认筛选状态的结果不得进入候选集。
4. 结果行使用快照中的列表/表格语义；DOM 回退为 `.result-table-list tbody tr`。逐条取题名、作者、来源、日期及详情链接，避免点击全页同名链接。

## 验证码真可见判断

若页面出现验证码候选元素，必须综合计算样式与边界框。只有 `display` 非 `none`、`visibility` 非 `hidden/collapse`、`opacity` 数值大于 0、宽高大于 0且与视口相交，才暂停让用户操作。元素存在但 `opacity:0` 属于隐藏占位，不阻塞检索。

## 逐篇与逐刊取证

1. 每个最终候选都打开详情页，核对页面题名、作者、期刊、年份/卷期页码或网络首发日期、摘要/关键词。记录 `verified_at`、页面所见 `verification_evidence`、实际 URL，以及能取得的 `filename`、数据库记录号或 DOI 等 `stable_cnki_id`。长 `v=` 会话 URL 只能作为当次证据，不能充当唯一稳定标识。
2. 对标为核心的每一种期刊，进入 CNKI 期刊导航后等待真实结果容器/结果计数出现，再按规范化后的完整刊名精确匹配。不要点击“最近浏览”“推荐期刊”或全页同名链接。打开详情后记录“被以下数据库收录”标签及页面证据。
3. 检索下一篇/下一刊前确认异步加载已完成，结果标题与当前查询一致；不一致就清空旧状态重查。

## 全文门禁

最终清单锁定后逐篇下载，优先 PDF、无 PDF 再尝试 CAJ。必须在下载管理与本地文件系统同时确认文件存在，才能写 `downloaded`。随后实际打开阅读全文，为正文将使用的观点记录页码/章节到 `claim_support_locations`，完成后才写 `fulltext_read=true`。

只看到题录或摘要、点击过下载按钮、保存了下载 URL，都不等于全文已下载或已读。任一最终文献被登录、权限、付费、验证码或证书错误阻断，立即停在来源门，不得先写“待核验稿”或基于摘要完成论文。
