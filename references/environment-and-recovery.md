# 环境与失败恢复

首次使用时先确认以下能力：网络；**用户可见的 Chromium 浏览器窗口**及其控制能力；可访问 CNKI 的浏览器会话；写入文件的工作目录；DOCX 创建与渲染工具。优先 Chrome，未安装或不可用时使用 Microsoft Edge。CNKI 不使用后台或无头浏览器作为默认执行面。

| 状态 | 可以完成 | 不能声称完成 | 恢复方式 |
| --- | --- | --- | --- |
| 无 CNKI 登录 | 题录检索、部分元数据核验 | 全文下载 | 用户在浏览器登录后继续 |
| 验证码出现 | 已完成内容和台账保存 | 验证码后的检索/下载 | 用户手动完成验证码后继续 |
| `ERR_CERT_*` 证书错误 | 已完成内容和台账保存 | 证书错误后的 CNKI 核验 | 在用户可见 Chrome 或 Edge 打开同一地址并由用户判断/处理；不得忽略证书错误或改用后台浏览器 |
| 无机构权限 | 题录核验与草稿 | 无权限全文已下载/已阅读 | 记录 `no_access`，由用户切换有权限会话 |
| 无可见 Chromium 浏览器控制 | 选题、结构、待核验草稿 | CNKI 已核验 | 让用户自行在 Chrome 或 Edge 打开 `https://kns.cnki.net/kns8s/search` 后回复“已打开”；不要用 `Start-Process`、`cmd start` 等命令行方式反复强行启动桌面浏览器，也不能退回后台浏览器声称已核验 |
| 无 DOCX 渲染 | 可生成 DOCX（若有生成器） | 已视觉验收 | 在具备渲染能力的 Codex 环境复验 |

绝不要求用户交出密码、Cookie 或验证码。不要尝试绕过登录、付费、验证码、证书警告或下载限制。
