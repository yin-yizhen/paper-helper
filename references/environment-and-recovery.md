# 环境与失败恢复

首次使用时先确认以下能力：网络；**用户可见且已连接到 Codex 的浏览器窗口**及其控制能力；可访问 CNKI 的浏览器会话；写入文件的工作目录；DOCX 创建与渲染工具。自动接管优先 Chrome 扩展会话或 Codex 应用内浏览器；Edge 可由用户手动使用，但不能仅凭已安装就假定可被 Codex 自动控制。CNKI 不使用后台或无头浏览器作为默认执行面。

开始检索前必须通过 [浏览器就绪门](browser-readiness.md)：能读取 CNKI 检索框或结果列表才算连接成功。用户说“已打开”或“继续”只表示用户希望继续，不构成连接或 CNKI 可用证据。

| 状态 | 可以完成 | 不能声称完成 | 恢复方式 |
| --- | --- | --- | --- |
| 无 CNKI 登录 | 无 | 任何论文或来源交付 | 用户在浏览器登录并通过浏览器就绪门后重新发起任务 |
| 验证码出现 | 仅保留已存在的内容和台账 | 验证码后的检索/下载及新论文交付 | 用户手动完成验证码并通过浏览器就绪门后重新发起任务 |
| `ERR_CERT_*` 证书错误 | 仅保留已存在的内容和台账 | 证书错误后的 CNKI 核验及新论文交付 | 在用户可见 Chrome 或 Edge 打开同一地址并由用户判断/处理；不得忽略证书错误或改用后台浏览器 |
| 无机构权限 | 无 | 任何论文或来源交付 | 由用户切换有权限会话后重新发起任务 |
| 无可见浏览器连接 | 无 | 题目、提纲、草稿、CNKI 核验及 Word 交付 | 让用户自行在 Chrome 或 Edge 打开 `https://kns.cnki.net/kns8s/search` 并建立可接管连接后重新发起任务；Edge 打开并不自动等于 Codex 已连接。不要用 `Start-Process`、`cmd start` 等命令行方式反复强行启动桌面浏览器，也不能退回后台浏览器声称已核验 |
| 无 DOCX 渲染 | 可生成内部预览 DOCX（若有生成器） | “最终版”“可直接提交”“已视觉验收” | 安装 LibreOffice/等价渲染后端后重新渲染；直接交付模式在此硬停止 |

绝不要求用户交出密码、Cookie 或验证码。不要尝试绕过登录、付费、验证码、证书警告或下载限制。

CNKI 访问不可用或不可接管时，采取**硬停止**：不得继续写作，也不得输出“待核验版”。仅告知阻塞原因和恢复条件；恢复后从浏览器就绪门重新开始。

“直接交付”还要求 DOCX 创建依赖和渲染后端均可用。CAJ 文件需要 CAJViewer 或可靠的文本读取能力；优先下载 PDF。浏览器、CNKI 账号/机构授权、验证码处理和 LibreOffice 属于外部环境，skill 只能预检、引导和阻断，不能自行安装授权或绕过访问控制。
